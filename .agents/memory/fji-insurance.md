# Feliciano Jiron Insurance site

- Single source of truth for service links is `window.JIRON_SERVICES` in `assets/site.js` (`file` field, e.g. `services/auto`). Header dropdown, mobile nav, footer, homepage cards, and the services overview all read from it — change links there, not per-page.
- Service subpages live at `site/services/<id>.html`, served at clean URLs `/services/<id>` (no `.html`): dev via Express `static({extensions:["html"]})`, prod via Vercel `cleanUrls:true`. They MUST include `<base href="/">` — without it the depth-0 relative links emitted by the shared `site.js` (which has no `../` awareness) break on these nested pages. **Why:** site.js is shared verbatim across all pages and emits root-relative-style hrefs; base href lets one shared engine work at any URL depth.
- Detail content for service subpages lives in `assets/service-details.js` (`window.JIRON_SERVICE_DETAILS`); subpages render via `assets/service-page.js` keyed off `body[data-service]`.
- **i18n pattern:** every translatable element carries a `data-es="..."` attribute holding the Spanish string; `window.jironSetLang(lang)` (exposed by site.js) swaps text. JS-injected markup MUST include `data-es`, then call `if (window.JIRON_LANG) window.jironSetLang(window.JIRON_LANG)` after inject. Spanish short labels = `JIRON_SERVICES[].es`; full names = `JIRON_SERVICE_DETAILS[].es`.
- **`data-es` and the default English BOTH render via `innerHTML`** (site.js `applyLang` does `el.dataset.en = el.innerHTML` then `el.innerHTML = …`), so HTML like `<strong>`, `<br>`, lists is allowed inside both — use real tags, not markdown. The only attribute-escaping helper is `esc()` (escapes `"` only); never put a literal `"` in a `data-es` value built via `data-es="${esc(...)}"`.
- Real per-service explainer copy was sourced from the agency's original site `jironinsuranceagency.com/our-services/<x>-insurance` (NOT the Farmers carrier links, which are generic). Lives in `JIRON_SERVICE_DETAILS[].sections` ([{h:[en,es], p?, list?}]). Do NOT use third-party brand names (e.g. USAA/Lemonade) in agency-voiced copy.
- **Cache-busting is manual and load-bearing:** assets are linked with `?v=N` query strings across every HTML page. When you edit a shared asset, bump its version in ALL pages that reference it (e.g. `styles.css?v=5`, `site.js?v=3`). Forgetting this serves stale JS/CSS to returning visitors.
- Shared component CSS (e.g. `.svc-grid/.svc-card`) belongs in the global `styles.css` (loaded everywhere), not page-specific sheets like `home.css`.
- **Logo/emblem sources:** `assets/img/logo-icon.png` is low-res (124px) and looks pixelated when scaled. The only high-res emblem is inside `assets/img/logo-full.png` (764px) but it sits on a gray metallic gradient AND its person figure is a knockout rendered in that same metallic gray (not white). To get a clean crisp emblem: crop the circle (exclude the baked-in wordmark), remove background, then force the desaturated figure/shadow pixels to white via a **saturation mask** (low-saturation→white, keeps the saturated blue disc + red ring), intersected with the original alpha. Header emblem is `<img class="blogo">` in `site.js` (NOT auto-cache-busted — add `?v=N` to the img src and bump `site.js?v=` so the new markup loads).
- **Cookie consent gates preference storage.** `applyTheme`/`applyLang` only write `jiron_theme`/`jiron_lang` to `localStorage` when `consentGiven()` is true (`jiron_cookie_consent === "accepted"`); Accept also persists current prefs + sets a `jiron_consent` cookie, Decline clears stored prefs + cookie. **Why:** consent must be functional, not cosmetic — declining means nothing is remembered across reloads. The consent flag itself (`jiron_cookie_consent`) is always stored (strictly-necessary) so the banner doesn't re-show.
- **Reveal-on-load "stutter" is a font-metrics reflow, not bad keyframes.** Fix = load fonts early (parallel `<link>`, never CSS `@import`) AND don't start animated reveals until `document.fonts.ready` (with a hard cap); keep above-fold text instant or the hero goes blank. **Why:** a font swap landing mid-tween reflows the glyphs and reads as jank.
- **PRODUCTION SERVES THIS SITE AS PURE STATIC `dist/public` — `server.mjs` does NOT run in prod.** The `.replit` `[deployment]` says `autoscale`/`router="application"`, but the deploy log shows `registered static handler ... publicDir=artifacts/fji-insurance/dist/public`. So any routing fix in `server.mjs` is DEV-ONLY and invisible to prod. **How to apply:** fix routing/URL problems at the file-structure level (so plain static serving works), then verify against a real static serve of `dist/public` (e.g. `python3 -m http.server` in that dir), never just the dev server. Prod requires a **re-publish** to pick up file changes.
- **A `foo.html` overview sharing a name with a `foo/` subpage dir — fix it with `foo/index.html`, NOT a server route or a redirect.** Move the overview to `foo/index.html` so EVERY static host serves it natively at `/foo/` (directory index); `/foo` then 301s to `/foo/` cleanly (single canonical redirect, no loop). Keep `<base href="/">` on it so its `assets/...` resolve from root at the `/foo/` URL. Update all links that pointed at `foo.html` (breadcrumbs, sitemap `<loc>`, canonical/og:url → `/foo/`, vercel redirect). **Why not a server route:** prod is static (see above) so routes never run there; the earlier route-only fix left prod serving the unstyled home page as a directory-fallback for `/foo/`. **Why not a redirect between `/foo`↔`/foo/`:** browsers permanently cache 301s and competing directions loop ("redirected too many times"). Dev `server.mjs` may additionally answer both with 200 for parity, but the file move is what fixes prod.
- **Blog post pages are GENERATED, not hand-edited — keep the generator's `<head>` template in sync.** `generate-posts.mjs` emits `site/posts/*.html`, and `build.mjs` re-runs it on every build. Any shared-asset change (font links, `?v=` cache versions, head tags) must be applied to BOTH the committed pages AND the `generate-posts.mjs` template, or the next build silently regenerates posts with stale refs. **Why:** a fix that only touches checked-in HTML is undone by the production build.
- **Global motion lives in `styles.css` + `site.js` (loaded on every page) — add site-wide transitions there, not per page.** **Why:** one shared stylesheet/script drives every page, so motion stays consistent and is cache-busted once. **How to apply:** (1) every new animation/transform MUST also be neutralized in the `prefers-reduced-motion` block at the end of `styles.css` — including hover-LIFT `transform`s, not just keyframes/transition timing (architect treats a missed hover lift as an a11y regression); (2) any JS-driven scroll ratio must clamp to `[0,1]` (`Math.max(0, Math.min(x,1))`) or elastic overscroll inverts it.
